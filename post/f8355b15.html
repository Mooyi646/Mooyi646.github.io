滑动窗口

一般思路：

一般用于：数组、字符串中求满足某条件的最大/小的连续子串【因为窗口是连续的】

  1. [left, right]为窗口，初始时left = right = 0
  2. right一直往右增加，直到不满足条件【也可能为直到条件满足】
  3. 缩小left直到条件再次满足【也可能为直到不满足条件】
  4. 重复2,3求最优

常见题目：

3. 无重复字符的最长子串

由题目可提取：

  • 条件：无重复字符
  • 子串
  • 最优：最长

由子串思考，一般用滑动窗口求解

  1. 窗口构造[left, right]，初始时都为0

  2. right增加，直到不满足条件，即出现重复字符X

  3. 缩小left直到再次满足条件，可知此时left应在X上一次出现的位置后方

    怎么知道X上一次出现的位置呢？

    Ans：使用数组或者map保存

    • 数组索引为字符X，值为出现位置，数组最大长度128即可
    • map key为字符X，值为位置

  4. 每次都保存子串的长度，有更长的则更新长度

  5. right到字符串尾则结束

所以依然考虑用滑动窗口解决，但是此时窗口长度固定。

    • 将p按字符大小排序，然后将窗口内的字符串也按大小排序，排序后相同则为异位词

一般分为两种：

  • 左右指针：两指针相向或者相背移动

快慢指针

一般思路为使用慢指针做有效值保存，使用快指针探测值是否有效

如：283. 移动零

15. 三数之和

  5. 不断调整固定的值（遍历），最后得出结果

回溯

解题思路：

本质其实为探索所有解中的可行解的算法。即：会穷举出所有解，选择满足条件的可行解。

本质其实相当于dsf + 剪枝

核心为for循环中的递归，在递归前选择元素，然后在递归调用之后撤销选择，即树形结构中回溯到上一层。

两个中间字段使用：

  • isUsed：用于判断是否已被用过，一般用于有唯一性的场景，即排列场景（[1,2,3], [3,2,1]为不同列表）
  • begin：用于去重，即不考虑begin索引之前的字符了，组合问题下使用[1,2,3], [3,2,1]为同一列表）

数位dp

一般用于统计某个范围的数，而这个范围一般都很大，难以直接暴力枚举求解，所以一般将其每位数拆分，一位一位地看。

如：判断在[0,x]范围中，满足某条件地数的个数

先做规约：

  • 0特殊判断，然后数位dp判断[1,x]
  • 不填前导0，即保证每个数的最高位为正数

填数过程，遵循以下：

  • 从高位往低位枚举数位
  • 在第i位填上了数，则其后面的数位都需填上数，保证数据不重复
  • 保证填入的数组成的数据不大于x

如：令x = 324783729，求[1,x]内有多少个数满足其数位上的数字之和为16

  1. 先看第一位：
    • 不填
    • 填：可以填1, 2, 3
  2. 再看第二位：
    • 不填
    • 填：
      • 第一位没填数：可以填[1,9]任意一个数
      • 第一位填数了：
        • 第一位填的1或2：可以填[0,9]任意一个数
        • 第一位填的3：可以填[0,2]任意一个数
  3. …
  4. 如处理到第七位：
    • 不填
    • 填：
      • 前面都没有填数：可以填[1,9]任意一个数
      • 前面填了：
        • 前面位数没满，即前面填的数没有六位：可以填[0,9]任意一位数
        • 前面数位满了：
          • 前面存在一位，其数字小于其对应的数字，即nums[i]：可以填[0,9]任意一位数
          • 前面所有数字都与对应数字相同：可以填[0,7]任意一位

规律分析总结：

第i位填数：

  • 前面都没有填数：[1,9]
  • 前面没填满或填满了但存在一位所填的数小于x中对应位数的数值：[0,9]
  • 前面所有数字都与x中对应数字相同：[0, nums[i]]

所以，可以引入变量limit来判断前面所填数字是否都与x中对应数字相同

则在填入第一个数时，即情况1，前面都没有填数时，对limit进行初始化

第二种情况下：limit = 0

第三种情况下： limit = 1

limit转移：

填完第i个数字，limit怎么转移到第i + 1位：

当前limit = 1且填入的数字为nums[i]，则下一位的limt = 1;

否则下一位的limit = 0;

特殊情况处理：

即最高位，第0位的处理

其填入的值的范围：[1, nums[0]]

其limit = 填值 == nums[i] ? 1 : 0

分析转移方程：使用dp[i][limit][sum]表示在前i位数字和为sum的数据个数

  • 前面都没有填数：[1,9]

  • 前面没填满或填满了但存在一位所填的数小于x中对应位数的数值：[0,9]

  • 前面所有数字都与x中对应数字相同：[0, nums[i]]
