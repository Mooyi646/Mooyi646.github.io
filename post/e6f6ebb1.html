Lambda表达式

也称闭包，允许将函数作为一个方法的参数，即把函数作为参数传递进方法中。

语法：

特征：

  • 可选类型声明：不需声明参数类型，编译器可以统一识别参数值
  • 可选的参数圆括号：一个参数无需定义圆括号，多个参数需要
  • 可选的大括号：只有一条语句，可以不使用大括号
  • 可选的返回关键字：只有一个表达式返回值则编译器会自动返回值，大括号需指定表达式返回了一个数值

注：

  • Lambda表达式主要用来定义行内执行的方法类型接口
  • Lambda表达式免去了使用匿名方法的麻烦，简单但强大的函数化编程能力

变量作用域

只能引用标记了final的外层局部变量/隐性的具有final语义，即不能在Lambda内部修改定义在域外的局部变量，否则会编译错误

Lambda表达式中不允许声明与局部变量同名的参数或局部变量。

双冒号:: 方法引用

::在Java8中被称作方法引用，提供了一种不执行方法的方法

一般使用lambda表达式会创建匿名方法，但有时只需调用已存在的方法，此时就可以用到方法引用。

使用场景：

  1. 静态方法引用（static method）语法：classname::methodname 例如：Person::getAge
  2. 对象的实例方法引用语法：instancename::methodname 例如：System.out::println
  3. 对象的超类方法引用语法： super::methodname
  4. 类构造器引用语法： classname::new 例如：ArrayList::new
  5. 数组构造器引用语法： typename[]::new 例如： String[]::new

Stream流

使用的是函数式编程模式，可以用来对集合进行链状流式操作。

Stream流不存储数据，但是可以检索和逻辑处理集合数据，包括筛选、排序、统计、计数，类似于SQL语句。

工作方式

  1. 创建流Object.stream()
  2. 对流进行中间操作：不加分号;，会再次返回一个流
  3. 对流进行终端操作：结束动作，一般返回void或非流结果

源数据：Collection、Array.

其方法参数都是函数式接口类型，所以一般和Lambda配合使用。

Stream流的处理顺序

中间操作特征：延时性

当且仅当存在终端操作时，中间操作才会被执行。

处理顺序：（垂直执行）随着链表垂直移动的，即完整执行一个元素的所有中间操作和终端操作后再执行下一个元素。

  • 垂直执行：先执行完某个元素的所有操作
  • 水平执行：先执行完所有元素的某个操作

注：sorted是水平执行的

原因：性能考虑，可以减少对每个元素的实际操作数。

        比如执行到某个元素整个流就停止执行了，则其后方元素就可以不被处理，即减少了其后方元素的中间操作执行过程。

如：共有5个元素，中间操作有3个，终端操作1个，为匹配到指定字符则返回true结束循环（anyMatch()）。假设第3个元素能成功匹配。

  • 处理顺序为所有元素执行完中间操作后再执行终端操作时：会执行$5 * 3$次中间操作，$3$次终端操作
  • 执行完一个元素所有操作再执行下一个元素：会执行$3*3$次中间操作，$3$次终端操作

通过适当调整中间操作的顺序，如先过滤再执行其他，可以有效提升性能。

数据流复用问题

Stream流是不能复用的，一旦调用终端操作，流就会关闭。如果此时再调用会抛出异常。

即一个Stream只能操作一次，不能断开，否则会报错。

克服以上限制：为每个终端操作创建新的流链。

常用方法

【stream操作类型】

  • 无状态：指元素的处理不受之前元素的影响。
  • 有状态：指该操作只有拿到所有元素之后才能继续下去。
  • 非短路操作：指必须处理所有元素才能得到最终结果。
  • 短路操作：指遇到某些符合条件的元素就可以得到最终结果，如 A || B，只要A为true，则无需判断B的结果。

其中：

  • 遍历/匹配：forEach / find / match
  • 筛选：filter
  • 聚合：max / min / count
  • 映射：map / flatMap
  • 归约：reduce 把一个流缩减成一个值，能实现对集合求和、求乘积和求最值操作。
  • 收集：collect
    • 归集：toList / toSet / toMap
    • 统计：count / averaging
    • 分组：partitioningBy / groupingBy
    • 接合：joining
    • 归约：reducing
  • 排序：sorted

forEach

功能：迭代流中的每个数据

map

功能：用于映射每个元素到对应的结果

filter

功能：设置条件过滤出元素

limit

作用：获取指定数量的流

如.limit(5)就为只获取5条

collect

终端操作，将流中元素转变为另一个不同的对象，如List, Set等。

参数：Collector(收集器)，一般直接使用内置的收集器即可

创建新的收集器：Collector.of()

主要使用java.util.stream.Collectors提供的静态方法。

如map部分的示例中的.collect(Collectors.toList());即为将流转为List

除了将流转化为集合元素外，还可以在流上执行聚合操作。

如计算平均数：.colllect(Collectors.averagingInt())

合并字符串：.colllect(Collectors.joining([前缀], 分隔符，[后缀]))，其中前缀与后缀可选
