HTML

HTML(Hypertext Makeup Language)，即超文本标记语言，主要负责网页内容的展示，大小写不敏感。

标记

一般以<标签名>…</标签名>的形式成对出现，自结束标签无需结束符

常见的标签

  • 文档声明[html5]：放在顶端

  • 网页根标签：<html>...</html>

  • <head>...</head>非用户可见，一般为告诉浏览器属性

    • head中的标题/浏览器头部标题：<title>text</title>

    • HTML文档元数据：<meta>,可说明编码类型，描述(discription)，作者(author)等等

      <meta charset="utf-8">

      <meta name="author" content="moo1">

    • 链接：<link>，定义文档与外部资源的关系，空元素，只包含属性，常用于来链接样式表。

      • 属性：

        1. herf：被链接文档的位置
        2. rel：当前文档与被链接文档之间的关系，必须
        3. type：type 属性规定被链接文档/资源的 MIME 类型。只有当设置了 href 属性时，才能使用该属性。<link> 标签常用的 MIME 类型是 “text/css”，它规定样式表。

        更多与link属性相关可点击查看。

组合选择符

后代选择器

选取某元素的后代元素，以空格 分离。

子元素选择器

只能选择作为某元素直接/一级子元素的元素，大于符号>分隔。

相邻兄弟选择器

选择紧接在另一元素后的元素，且二者有相同父元素，用+分隔。

后续兄弟元素

后续兄弟选择器选取所有指定元素之后的相邻兄弟元素。用~隔开。

一般用于同一样式需用于很多页面，通过mystyle.css文件改变站点整体样式【页面使用<link>标签链接到样式表，该标签位于文档头部<head>中】

变量

使用关键字var定义变量，var a=1;

  • 变量声明var name;,此时name的值为undefined

变量作用域

  • 函数体外声明的变量为全局变量，函数内声明的变量为局部变量(var)

  • 在块中声明var变量，其在块外依然可被访问

  • 使用let关键字可实现只在块内有效

  • 重新定义变量

    • var：可在任意地方修改,
    • let：
    • 相同作用域/块级作用域中，不能用let关键字重置var/let声明的变量

    • 相同作用域/块级作用域中，不能用 const关键字重置var/let/const声明的变量

  • 使用var声明的全局变量属于window对象.

  • 使用let声明的全局变量不属于window对象.

  • var变量可以先使用再声明(作为全局变量，可删除,严格模式下不允许)，let变量必须声明才能使用。

正则表达式(Regular Expression/regex/regexp/RE)

  • 正则表达式是由一个字符序列形成的搜索模式。

  • 当你在文本中搜索数据时，你可以用搜索模式来描述你要查询的内容。

  • 正则表达式可以是一个简单的字符，或一个更复杂的模式。

  • 正则表达式可用于所有文本搜索和文本替换的操作。

语法

/正则表达式主体/修饰符(可选)

主体用于检索，不区分大小写

  • 修饰符
    • i： 执行对大小写不敏感的匹配。
    • g：执行全局匹配（查找所有匹配而非在找到第一个匹配后停止）。
    • m：执行多行匹配。

使用字符串方法

  • search()

    检索字符串中指定的子字符串，或检索与正则表达式相匹配的子字符串，并返回子串的起始位置。可使用字符串作参数（自动转化为正则表达式）。

  • replace()

    使用正则表达式且不区分大小写将字符串中的A替换为B，可使用字符串作参数

JSON

JSON(JavaScript Object Notation)，使用JS语法，但是独立于语言，JSON是轻量级的文本数据交换格式。

语法规则

  • 数据为键值对("key":value)，key必须是字符串，必须在双引号中

    访问对象值

    • obj.key
    • obj[“key”]

  • 数据由,分隔

  • 大括号{}保存对象，方括号[]保存数组

JSON字符串转换为JS对象

使用JSON.parse(text,reviver)将接受的数据(字符串)转化为JS对象

  • text：必需，一个有效的JSON字符串
  • reviver：可选，一个转换结果的函数， 将为对象的每个成员调用此函数。
  • 检验JSON格式是否合法
Atten
  1. JSON不能存储Date对象，如果需要存储 Date 对象，需要将其转换为字符串。之后再将字符串转换为 Date 对象。

​ 可以使用reviver参数，一个转换结果的函数，对象的每个成员调用此函数。

  1. JSON 不允许包含函数，但以将函数作为字符串存储，之后再将字符串转换为函数。

    不建议使用函数

JS对象转换为字符串

使用JSON.stringify(value,replacer,space)将 JavaScript 对象转换为字符串。

  • value:必须， 要转换的 JavaScript 值（通常为对象或数组）。

  • replacer:可选。用于转换结果的函数或数组。

    如果 replacer 为函数，则 JSON.stringify 将调用该函数，并传入每个成员的键和值。使用返回值而不是原始值。如果此函数返回 undefined，则排除成员。根对象的键是一个空字符串：””。

    如果 replacer 是一个数组，则仅转换该数组中具有键值的成员。成员的转换顺序与键在数组中的顺序一样。当 value 参数也为数组时，将忽略 replacer 数组。

  • space:可选，文本添加缩进、空格和换行符，如果 space 是一个数字，则返回值文本在每个级别缩进指定数目的空格，如果 space 大于 10，则文本缩进 10 个空格。space 也可以使用非数字，如：\t。

Atten
  1. JSON 不能存储 Date 对象。JSON.stringify() 会将所有日期转换为字符串。
  2. JSON 不允许包含函数，JSON.stringify() 会删除 JavaScript 对象的函数，包括 key 和 value。
